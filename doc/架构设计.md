# 项目架构设计

## 一、项目定位

**comp-vue-lib** 是一个基于 **Vue 3 + TypeScript + Vite** 的企业级 UI 组件库，核心亮点是 **BaseTable 五种渲染模式**（从 DOM 到 Canvas/Skia WASM），适配从小数据到超大数据集的表格场景。

---

## 二、技术栈

| 层面       | 技术                                         |
| ---------- | -------------------------------------------- |
| 框架       | Vue 3.5 + Composition API (`<script setup>`) |
| 类型       | TypeScript 5.6                               |
| 构建       | Vite 5（开发 + 库模式打包）                  |
| UI 基座    | Element Plus 2.8                             |
| 高性能渲染 | Canvas 2D / CanvasKit（Skia WASM）           |
| 包管理     | pnpm 10                                      |
| 代码质量   | ESLint + Prettier + Stylelint                |
| 类型声明   | vite-plugin-dts（rollupTypes 聚合输出）      |

## 四、核心架构：BaseTable 五模式

BaseTable 是本项目的核心组件，通过 `mode` prop 切换五种渲染策略：

```
BaseTable.vue（外壳）
  ├── BaseTableElement.vue      → el-table 封装（功能最全）
  ├── BaseTableVirtual.vue      → el-table-v2 虚拟滚动
  ├── BaseTableCanvas.vue       → Canvas 2D 视口重绘
  ├── BaseTableCanvasTile.vue   → Canvas 整表预渲染 + 裁切
  └── BaseTableSkiaWasm.vue     → CanvasKit GPU 加速渲染
```

### 设计思路

1. **统一接口**：外壳 `BaseTable.vue` 接收统一的 `columns` / `tableData` / `rowKey` 等 props，内部 `<KeepAlive>` 按 mode 分发
2. **统一主题**：`configInjection` 提供全局配置，DOM 模式通过 `tableSurfaceCssVars()` 生成 CSS 变量注入，Canvas 模式通过 `getTableSurface()` 获取 JS 常量绘制，颜色均从 `LibThemeConfig` 自动派生
3. **渐进增强**：element 模式能力最全（支持 slot、排序、展开行等），canvas 系模式逐步覆盖交互（checkbox hover、tooltip、滚动条等）
4. **可选依赖**：`canvaskit-wasm` 标记为 `peerDependenciesMeta.optional`，仅 skia-wasm 模式需要

### 各模式适用场景

| 模式          | 适用场景               | 特点                                 |
| ------------- | ---------------------- | ------------------------------------ |
| `element`     | 常规数据量（<1000 行） | 能力最全，支持所有 Element Plus 特性 |
| `virtual`     | 万级行数据             | DOM 仍可控，虚拟滚动                 |
| `canvas`      | 超大数据、结构简单     | 单 Canvas，视口内重绘                |
| `canvas-tile` | 超大数据               | 整表预渲染，滚动裁切视口             |
| `skia-wasm`   | GPU 加速需求           | CanvasKit 渲染，需加载 wasm          |

### 共享工具层

- `canvasDraw.ts` — Canvas 2D 通用绘制函数
- `useCanvasScrollbar.ts` — 自定义滚动条
- `useCanvasTooltip.ts` — hover 显示完整文本
- `useCanvasCheckboxHover.ts` — checkbox 交互
- `useCanvasSlotPopup.ts` — Canvas 模式弹窗
- `useBaseTableSelection.ts` — 行选择逻辑
- `column.ts` — 列宽计算（五模式对齐）
- `tableHitTest.ts` — Canvas 点击命中检测
- `selectionKeys.ts` — 选择键管理
- `skiaTypeface.ts` — Skia 字体加载
- `canvaskitLocate.ts` — CanvasKit WASM 资源定位

## 五、配置注入系统

### 架构

配置在初始化时固化，运行时通过 `saveConfig()` 更新内存 + localStorage + CSS 变量。

```
createCompLib(userConfig?, persist?)
    ↓
resolveConfig(defaults + userConfig + localStorage)
    ↓
┌─────────────────────────────────────────────────┐
│ app.provide(LIB_CONFIG_KEY, resolved)           │ → 组件通过 useLibConfig() 注入
│ applyCssVars(resolved)                          │ → CSS 变量注入 :root
└─────────────────────────────────────────────────┘
    ↓
saveConfig(newConfig)
    ↓
┌─────────────────────────────────────────────────┐
│ Object.assign(resolved, newConfig)              │ → 同步更新内存
│ applyCssVars(resolved)                          │ → 刷新 CSS 变量
│ writeToStorage(resolved)                        │ → 持久化到 localStorage
└─────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────┐
│ DOM 模式: tableSurfaceCssVars(config)           │ → 从 theme 自动派生表格颜色
│ Canvas 模式: getTableSurface(config)            │ → JS 常量直接绘制
│ ConfigProvider: 嵌套覆盖 + 局部 CSS 变量        │
└─────────────────────────────────────────────────┘
```

### 配置优先级（从低到高）

1. `configDefaults.ts` 代码默认值
2. `createCompLib(userConfig)` 初始化传入
3. `localStorage` 持久化的值
4. `ConfigProvider` 组件级覆盖
5. 组件 `props.theme` 局部覆盖

### 对外配置接口

| 接口             | 说明                                                   |
| ---------------- | ------------------------------------------------------ |
| `LibThemeConfig` | 全局主题（品牌色、文字色、背景色、边框色、字体、圆角） |
| `LibTableConfig` | 表格布局（行高、表头高度、字号、字重、列宽）           |
| `LibConfig`      | 消费者传入（所有字段可选）                             |
| `PersistOptions` | 持久化选项（enabled + key）                            |

### 表格颜色派生规则

表格颜色不对外暴露，从 `LibThemeConfig` 自动派生：

| 表格属性          | 派生来源               | EP 对应变量                    |
| ----------------- | ---------------------- | ------------------------------ |
| surfaceBg         | `theme.bgCard`         | `--el-table-bg-color`          |
| headerBg          | `theme.bgPage`         | `--el-table-header-bg-color`   |
| rowBase           | `theme.bgCard`         | `--el-table-tr-bg-color`       |
| rowStripe         | `theme.bgSubtle`       | `--el-fill-color-lighter`      |
| rowHoverBg        | `theme.bgPage`         | `--el-table-row-hover-bg-color`|
| borderColor       | `theme.borderColor`    | `--el-table-border-color`      |
| checkboxCheckedBg | `theme.colorPrimary`   | `--el-color-primary`           |
| textPrimary       | `theme.textRegular`    | `--el-table-text-color`        |
| textHeader        | `theme.textSecondary`  | `--el-table-header-text-color` |

### CSS 变量命名

- 全局前缀：`--comp-*`（如 `--comp-color-primary`、`--comp-bg-page`）
- 表格局部：`--crud-bt-*`（如 `--crud-bt-row-height`）
- Element Plus 同步：`--el-color-primary`、`--el-bg-color` 等（随 `saveConfig` 刷新）
- **default 主题**：主色 / 功能色 / 圆角对齐 Element Plus 默认风格，表格 surface 派生色完全匹配 EP 原生表格

### Element Plus 国际化

- 导出 `elementPlusLocale`（zh-cn）
- Demo 入口 `app.use(ElementPlus, { locale })`
- `ConfigProvider` 内嵌 `ElConfigProvider` 包裹子树

### 消费者使用

```ts
import { createCompLib } from "comp-vue-lib";

const lib = createCompLib(
  { theme: { colorPrimary: "#6366f1" } },
  { enabled: true }, // 开启 localStorage 持久化
);
app.use(lib);

// 保存配置到 localStorage，刷新后生效
lib.saveConfig({ theme: { colorPrimary: "#10b981" } });
```

## 六、构建与分发

| 配置                 | 用途                                                             |
| -------------------- | ---------------------------------------------------------------- |
| `vite.config.ts`     | 开发服务器，root 设为 `examples/`，构建输出到 `dist-playground/` |
| `vite.lib.config.ts` | 库模式打包，输出 ESM + CJS + 类型声明 + sourcemap                |

### 打包策略

- **多入口构建**：主入口（组件）+ util 子模块 + type 子模块，支持精确 tree-shaking
- **外部化**：vue / element-plus / @element-plus/icons-vue / lodash-es / canvaskit-wasm 不打入包内
- **双格式**：每个入口输出 `.mjs`（ESM）+ `.js`（CJS）
- **类型声明**：`vite-plugin-dts` 为每个模块生成独立 `.d.ts`
- **CSS 合并**：`cssCodeSplit: false`，所有组件样式输出到 `style.css`
- **sideEffects**：`package.json` 标记仅 CSS 有副作用，bundler 可安全 shake

### 导出接口

```ts
// 组件（主入口）
import { createCompLib, BaseTable, BaseCrud, CodeBlock, WidgetTabs } from "comp-vue-lib";

// 工具函数（/util 子路径）
import { rgbaToHex, debounce, usePermission } from "comp-vue-lib/util";
import { rgbaToHex } from "comp-vue-lib/util/color";

// 类型（/type 子路径）
import type { TagProps, BaseTableColumn } from "comp-vue-lib/type";

// 样式
import "comp-vue-lib/style.css";
```

### package.json exports 映射

```json
{
  ".": { "import": "./dist/index.mjs", "require": "./dist/index.js", "types": "./dist/index.d.ts" },
  "./util": { "import": "./dist/util/index.mjs", "require": "./dist/util/index.js", "types": "./dist/util/index.d.ts" },
  "./util/*": { "import": "./dist/util/*.mjs", "require": "./dist/util/*.js", "types": "./dist/util/*.d.ts" },
  "./type": { "import": "./dist/type/index.mjs", "require": "./dist/type/index.js", "types": "./dist/type/index.d.ts" },
  "./type/*": { "import": "./dist/type/*.mjs", "require": "./dist/type/*.js", "types": "./dist/type/*.d.ts" },
  "./style.css": "./dist/style.css"
}
```

## 七、三端适配策略

通过 SCSS 断点变量统一管理：

| 端     | 断点           | 变量             |
| ------ | -------------- | ---------------- |
| PC 端  | > 1024px       | 默认布局         |
| 平板端 | 768px ~ 1024px | `$doc-bp-tablet` |
| 手机端 | < 768px        | `$doc-bp-mobile` |

### 适配要点

- 交互元素点击区域不小于 32×32px
- 布局在窄屏下不溢出
- Canvas 模式支持触摸滚动（惯性 + 方向判断）
- 抽屉组件移动端全屏适配
- 滚动穿透锁定
- 演示站导航：平板/移动端使用底部 Tab 栏替代侧边栏，释放内容区宽度

## 八、组件设计模式

1. **类型先行**：类型声明统一放在 `src/type/`，组件目录内不保留 `types.ts`
2. **配置驱动**：BaseSearch / BaseTable 通过 JSON 配置数组描述字段/列
3. **CRUD 联动**：BaseCrud 组合 BaseSearch + BaseTable + BaseColumnSetting + BaseSearchDrawer
4. **列设置入口**：editColumn 列固定最右，不参与列设置面板排序
5. **组合式函数**：复杂逻辑抽取为 `use*` hooks（如 `useCanvasScrollbar`）
6. **插件式安装**：`createCompLib()` 工厂模式，初始化固化配置，`saveConfig()` 更新内存 + localStorage + CSS 变量
7. **分层导出**：组件从主入口导出，工具函数从 `/util` 子路径导出，类型从 `/type` 子路径导出
8. **配置派生**：业务细节颜色从统一主题自动派生，减少消费者配置负担
9. **组件复用**：相同功能提取为通用组件（如 CodeBlock、WidgetTabs），Demo 站通过薄包装层复用
10. **CSS 变量主题化**：基础组件通过 CSS 变量暴露样式控制，消费方可自定义
